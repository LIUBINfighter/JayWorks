遵循官方文档主要基线（Google Developer Style + Microsoft Style 作为补充），并针对插件内置文档场景做裁剪：

## 术语建议

- 使用“笔记”指代 Markdown 文件。
- 使用“文件”指代非 Markdown 资源。
- 使用“文件夹”而非“目录”。

## 交互表述

- 多级操作路径使用：**设置 → 社区插件 → 已安装**。
- 避免方向性描述（左侧 / 右侧）除非必要。

## 指令语气

- 标题与步骤使用祈使句（“创建配置文件” 而非 “创建配置文件的方式”）。

## 图像与图标

- 优先 SVG，其次 PNG。
- 控制宽度（默认展示宽度即可，避免超宽滚动）。
